Análise de desempenho de métodos de requisição do Google recaptcha.

Esta é uma página que é composta por diversos gráficos, sobre esses gráficos foi feita uma análise do desempenho comparando dois métodos de requisição, visando definir se havia indicativos que poderia haver ganhos de performance ao alterar a função apex_web_service.make_rest_request, para a função de mais baixo nível.

Existe um pacote que faz uma requisição ao Google recaptcha para renderizar o elemento e também validar o desafio de verificação de usuário humano, isso é aplicado em página públicas visando evitar o acesso indevido de robôs.

Esta verificação gerou uma certa demora em certos momentos, e surgiu a tarefa de analisar esses tempos, assim como, avaliar como solução uma função alternativa para verificar o desafio.

Foi criado um procedimento que coletava 1000 amostras de requisição para cada função inserindo a o tempo de resposta em uma tabela, e a partir dessa tabela foram extraidos os dados para os gráficos dessa tela.

A partir da visualização das médias e desvios padrão dos tempos foi possivel concluir que não havia uma real vantagem de mudar a implementação, havia uma melhora de alguns milisegundos por requisição, mas também piorava a consistência, como pode ser vista pela análise do desvio padrão mostra.

Através dessa aplicação foi possivel treinar habilidades de análise de dados e estatística, além de exercitar os conhecimentos de criação de gráficos em Apex.
